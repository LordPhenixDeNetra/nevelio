⚡ Installation

Prérequis

🦀
Rust stable ≥ 1.75
Installer via rustup update stable
🌐
Accès réseau à la cible
HTTPS recommandé, HTTP supporté
📋
Autorisation écrite
Obligation légale dans toutes les juridictions

Compilation

bash
git clone https://github.com/your-org/nevelio.git
cd nevelio
cargo build --release

# Le binaire est dans target/release/nevelio
./target/release/nevelio --version

Installation globale (optionnel)

bash
# Copier dans $PATH
cp target/release/nevelio ~/.local/bin/

# ou via cargo
cargo install --path crates/cli

# Vérification
nevelio --version   # nevelio 0.1.0
nevelio --help

Persistance automatique

Après acceptation, un marqueur est écrit dans ~/.config/nevelio/legal_accepted. Les lancements suivants ne demandent plus confirmation.

bash
cat ~/.config/nevelio/legal_accepted

⚙️ Configuration — nevelio init

Génère un fichier .nevelio.toml commenté dans le répertoire courant :

bash
cd mon-projet
nevelio init
# Crée .nevelio.toml avec tous les champs documentés

🔍 Commande scan

Cible et spec

FlagDescriptionExemple
--target URLURL de base de l'API--target https://api.example.com
--url URLAlias de --target--url https://api.example.com
--spec FILE|URLSpec OpenAPI JSON ou YAML--spec ./openapi.yaml
bash
# Avec spec OpenAPI — couverture maximale
nevelio scan --target https://api.example.com --spec ./openapi.yaml --accept-legal

# Sans spec — découverte automatique des endpoints
nevelio scan --target https://api.example.com --accept-legal

Profils de scan

🌿 stealth
1 req simultanée · 2 req/s
Production sensible

Évite les alertes IDS/WAF. Scan lent mais discret.

⚖️ normal
5 req simultanées · 10 req/s
Défaut recommandé

Équilibre vitesse/discrétion. Idéal pour le staging.

⚡ aggressive
20 req simultanées · 50 req/s
Lab uniquement

Scan rapide. Ne pas utiliser en production.

bash
nevelio scan --target https://api.example.com --profile stealth   --accept-legal
nevelio scan --target https://api.example.com --profile normal    --accept-legal
nevelio scan --target https://api.example.com --profile aggressive --accept-legal

Sélection de modules

Par défaut, tous les modules sont exécutés. Pour restreindre :

bash
# Un seul module
nevelio scan --target https://api.example.com --module auth --accept-legal

# Plusieurs modules (espace-séparés)
nevelio scan --target https://api.example.com \
             --module auth injection access-control \
             --accept-legal

Authentification et proxy

bash
# JWT Bearer token
nevelio scan --target https://api.example.com \
             --auth-token "Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
             --accept-legal

# Basic auth (user:password encodé en base64)
nevelio scan --target https://api.example.com \
             --auth-token "Basic dXNlcjpwYXNz" \
             --accept-legal

# Via proxy Burp Suite
nevelio scan --target https://api.example.com \
             --proxy http://127.0.0.1:8080 \
             --accept-legal

Sortie et rapports

bash
# HTML (défaut) — findings.json + report.html
nevelio scan --target https://api.example.com --accept-legal

# SARIF pour GitHub Security
nevelio scan --target https://api.example.com \
             --output sarif \
             --out-dir ./sarif-results \
             --accept-legal

# Bloquer CI sur findings HIGH ou CRITICAL
nevelio scan --target https://api.example.com \
             --fail-on high \
             --accept-legal
echo $?   # 1 si HIGH/CRITICAL trouvé

Module auth — Détail

HIGH Missing Authentication CWE-306 ▶

Teste chaque endpoint sans header Authorization. Un code 200/201 indique une authentification manquante.

Exemple de finding :
json
{
  "title": "Missing Authentication",
  "severity": "High",
  "cwe": "CWE-306",
  "endpoint": "GET /api/v1/users",
  "recommendation": "Implémenter un middleware d'authentification obligatoire."
}
CRITICAL JWT Algorithm None CWE-327 ▶

Envoie un JWT avec "alg": "none" et signature vide. Si le serveur l'accepte, il ne valide pas la signature.

payload forgé
eyJhbGciOiJub25lIiwidHlwIjoiSldUIn0.eyJzdWIiOiIxMjM0In0.

Remédiation : Rejeter explicitement l'algorithme "none". Utiliser RS256 ou ES256.

HIGH JWT Weak Secret CWE-330 ▶

Tente de valider la signature HMAC-SHA256 avec des secrets faibles communs : secret, password, 123456, changeme…

HIGH JWT Claims Manipulation CWE-269 ▶

Modifie les claims role → admin, is_admin → true, et re-signe avec le secret faible découvert.

MEDIUM Basic Auth Brute Force CWE-521 ▶

Sur les endpoints avec WWW-Authenticate: Basic, teste un dictionnaire de couples user:password communs.

Module injection — Détail

CRITICAL SQL Injection CWE-89 ▶

4 techniques sur chaque paramètre :

TechniquePayloadDétection
Time-based' OR SLEEP(5)--Délai ≥ 4s
Error-based' OR 1=1--Message SQL dans réponse
Union-based' UNION SELECT NULL--Données injectées
Boolean-based' AND 1=1-- vs ' AND 1=2--Contenu différent
HIGH NoSQL Injection CWE-943 ▶
json — opérateurs MongoDB
{"username": {"$gt": ""}, "password": {"$gt": ""}}
{"username": {"$regex": ".*"}}
HIGH Server-Side Template Injection (SSTI) CWE-94 ▶
MoteurPayloadRésultat
Jinja2 / Twig{{7*7}}49
FreeMarker${7*7}49
Pebble{{7*'7'}}7777777
CRITICAL Command Injection CWE-77 ▶
payloads
; id
| whoami
`id`
$(id)
& ping -c 1 127.0.0.1

Détection : délai (sleep) ou présence de uid= dans la réponse.

Module access-control — Détail

HIGH IDOR numérique / UUID CWE-639 ▶

Substitue les IDs dans l'URL par des valeurs voisines (/users/42 → /users/41) ou des UUIDs aléatoires. Compare les réponses pour détecter un accès à des ressources d'autres utilisateurs.

HIGH BFLA — Broken Function Level Authorization CWE-285 ▶

Teste les endpoints /admin/*, /management/*, /internal/* avec le token utilisateur standard. Un code 200 indique une autorisation mal configurée par rôle.

HIGH Mass Assignment CWE-915 ▶
json — champs injectés
{"name": "John", "role": "admin", "is_admin": true, "credits": 999999}

Module graphql — Détail

MEDIUM Introspection exposée CWE-200 ▶
graphql
{ __schema { types { name fields { name } } } }

Si le serveur répond avec le schéma complet, la structure interne est exposée à un attaquant.

MEDIUM Depth DoS CWE-400 ▶
graphql
{ user { friends { friends { friends { friends { id name } } } } } }

Sans limite de profondeur, le serveur peut subir une surcharge CPU/mémoire.

Module business-logic — Vérifications

HIGH Rate Limiting absent CWE-770 ▶

Envoie 20 requêtes rapides et vérifie l'absence de réponse 429 Too Many Requests. Sans rate limiting, login/OTP/reset sont vulnérables au brute force.

MEDIUM X-Forwarded-For bypass CWE-290 ▶
headers injectés
X-Forwarded-For: 127.0.0.1
X-Real-IP: 127.0.0.1
CF-Connecting-IP: 192.168.1.1
HIGH Race Condition CWE-362 ▶

Envoie simultanément 10 requêtes identiques (achat, réservation, coupon) et détecte si plusieurs répondent avec succès alors qu'une seule devrait l'être.

🤖 Suggestions IA via Claude

Nevelio peut générer des recommandations de remédiation détaillées pour chaque finding, via l'API Claude d'Anthropic.

ℹ️
Modèle utilisé : claude-haiku-4-5-20251001. Sortie : <out-dir>/ai_suggestions.md
bash
export ANTHROPIC_API_KEY=sk-ant-api03-...

nevelio scan --target https://api.example.com \
             --ai-suggestions \
             --out-dir ./results \
             --accept-legal

# Lire les suggestions générées
cat ./results/ai_suggestions.md
⚠️
Si ANTHROPIC_API_KEY est absent, Nevelio affiche un avertissement avant le scan et continue normalement sans les suggestions.

📄 Commande report / convert

Convertit un findings.json existant en un autre format, sans relancer le scan.

bash
# Générer un rapport HTML depuis un JSON existant
nevelio report --input ./results/findings.json \
               --format html \
               --out-dir ./rapports

# Alias "convert" — fonctionnement identique
nevelio convert --input findings.json --format sarif --out-dir ./security

🔄 Intégration CI/CD

✅
Combinaison recommandée pour tout pipeline CI : --accept-legal --no-tui --no-color --fail-on high
.github/workflows/security.yml
name: API Security Scan

on:
  pull_request:
  push:
    branches: [main]

jobs:
  nevelio-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install Rust
        uses: dtolnay/rust-toolchain@stable

      - name: Build Nevelio
        run: cargo build --release --bin nevelio

      - name: Run API Security Scan
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          ./target/release/nevelio scan \
            --target ${{ vars.API_STAGING_URL }} \
            --spec openapi.yaml \
            --output sarif \
            --out-dir sarif-results \
            --fail-on high \
            --accept-legal \
            --no-tui \
            --no-color \
            --ai-suggestions

      - name: Upload SARIF to GitHub Security
        uses: github/codeql-action/upload-sarif@v3
        if: always()
        with:
          sarif_file: sarif-results/report.sarif

      - name: Upload AI Suggestions
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: ai-suggestions
          path: sarif-results/ai_suggestions.md
.gitlab-ci.yml
api-security-scan:

🎯 Scénarios pratiques

Audit rapide sans spec OpenAPI

Idéal pour une première prise en main ou une cible sans spec disponible.

bash
nevelio init   # crée .nevelio.toml
# éditer .nevelio.toml selon la cible

nevelio scan --target https://api.example.com \
             --profile stealth \
             --accept-legal

open report.html   # macOS
xdg-open report.html   # Linux

Audit professionnel complet

bash
TOKEN="Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."

nevelio scan \
  --target https://staging.api.example.com \
  --spec ./openapi.yaml \
  --profile normal \
  --auth-token "$TOKEN" \
  --out-dir ./pentest-$(date +%Y%m%d) \
  --output html \
  --fail-on high \
  --accept-legal

open ./pentest-$(date +%Y%m%d)/report.html

Audit avec interception Burp Suite

bash
# Burp Suite en écoute sur 127.0.0.1:8080
nevelio scan \
  --target https://api.example.com \
  --module auth injection \
  --proxy http://127.0.0.1:8080 \
  --no-tui \
  --verbose \
  --accept-legal
# Les requêtes apparaissent dans l'historique Burp

Scan avec suggestions IA et multi-format

bash
export ANTHROPIC_API_KEY=sk-ant-api03-...

nevelio scan \
  --target https://api.example.com \
  --spec ./openapi.yaml \
  --ai-suggestions \
  --out-dir ./results \
  --accept-legal

# Résultats dans ./results/ :
# ├── findings.json       (données brutes)
# ├── report.html         (rapport interactif)
# └── ai_suggestions.md   (remédiation IA)

# Générer un SARIF en plus
nevelio convert --input ./results/findings.json \
                --format sarif \
                --out-dir ./results

Reprise après interruption

bash
# Scan initial — interrompu par ^C ou crash
nevelio scan --target https://api.example.com \
             --spec ./openapi.yaml \
             --out-dir ./results \
             --accept-legal

# Reprise — saute les modules déjà complétés
nevelio scan --target https://api.example.com \
             --spec ./openapi.yaml \
             --out-dir ./results \
             --resume \
             --accept-legal

🌐 Internationalisation

Nevelio supporte 3 langues — français (fr), anglais (en), espagnol (es). Tous les messages CLI, le TUI, le disclaimer légal, les suggestions IA et le rapport HTML sont traduits automatiquement.

Détection automatique

La langue est résolue dans cet ordre de priorité :

PrioritéSourceExemple
1Flag --langnevelio --lang es scan …
2Variable NEVELIO_LANGNEVELIO_LANG=fr nevelio scan …
3Variable système $LANGUAGE / $LANGLANG=fr_FR.UTF-8 → fr
4Défauten

La normalisation est automatique : fr_FR.UTF-8, fr-FR et fr donnent tous fr.

Exemples

LANG=fr_FR.UTF-8 nevelio scan --target https://api.example.com --dry-run
# Cible       : https://api.example.com

Rapport HTML localisé

Le rapport HTML généré (report.html) hérite de la langue du scan. Tous ses labels (Cible / Target / Objetivo, Description, Recommandation…) sont traduits automatiquement.

Suggestions IA localisées

Avec --ai-suggestions, le prompt envoyé à Claude est rédigé dans la langue sélectionnée — les suggestions reçues sont donc dans la même langue.

nevelio --lang es scan --target https://api.example.com \
  --ai-suggestions --accept-legal
# → Les suggestions IA seront en espagnol
